Box Model

And this Four-Minute web design demo is a good, and quick demonstration of why positioning makes good design.

CSS/HTML has a visual design metaphor called the "box model" The first thing to know about the Box Model metaphor is that all html elements can be visualized as a "box". A box has content (with a width and height), border, padding, and margins. And the size of these can be set with CSS to determine how much space each element takes up.

Some elements in HTML are "block" level elements, some are "inline-block" level elements and some are "inline". These three designations control how much space each item takes up on the page by default (no matter how "big" it is) and whether it is displayed to its full width and height. The heading above is a "block" level element by default. "Block" level elements take up the full width of the page and structure the page flow. h1 and p and div are all block level. If they have a width and height designated they will take up this full space. "Inline-block" level elements are displayed to any designated width and height, but stay "inline" in the page flow, meaning elements can appear "alongside" them. "Inline" level elements appear inline in the text. Like this element or this element.This Link to example.com is also an inline element. Block, inline-block, and inline are "Display" attributes. You can always give these elements room by designating margins, padding and border. And you can change the display of any element with the "display" property, and by desgintating it "inline", "block" or "inline-block".

SPACE!

One of the first things to do to start a good web page is to give everything a bit of room. You may want to give the body element a bit of padding so all of your content has some room on the page. Then build from the top to the bottom, positioning each element, giving it space, and determining how much width each will take up. Content and the Margin and Padding attributes usually account for the height of an element, and the page scrolls down as elements take up space.

You can also use things like "margin" to help center and position things on a page. "Inline" and "inline-block" elements can be positioned with "text-align", as they are based on the line for alignment. "Block" level elements can be given margin values of any number for top and bottom and then have left and right set to "auto" to be positioned in the center of any page block. The above image of the box model is an example of this. Check the inline styles.

Finally, remember you can use em values, pixel values or percentages to set height and width. Using percentage lets the element size to a percent of it's parent element. This can be useful and will come in handy when creating responsive websites and creating full-screen designs.

Many websites tend to be set up in an F-Pattern, reading from top to bottom and from left to right. This is a web convention, and normalizes left-to-right reading. *Note: there are some good reasons to break this convention: to pique visual interest or build a more engaging design (i.e. prevent the user from quickly scanning before moving on). The placement of links and clickable items may also be something you want to think about formatting for differently-sized screens and for navigation by a thumb touch rather than a mouse-click (this is called "responsive design" and we'll return to it later).

Position

There are also many options for setting the Positioning and Floating of elements. Elements may have a "Fixed", "Absolute" or "Relative" positioning. Elements are "static" by default and fit within the flow of the page, and this might be behavior you want. "Fixed" position elements are positioned relative to the viewport. You can specify "top", "left", "bottom" and/or "right" as attributes and set this position in pixel values. These elements will scroll along with the page, and this is often used for the positioning of menus, footers, or "special" content that you want to have consistently available in the viewport or draw attention to. "Absolutely" positioned elements are positioned relative to their nearest ancestor (parent element). If there is no parent they will behave like fixed positioned elements and scroll with the page. "Relatively" positioned elements can be given a specific position and sit relative to this, by again specifying "top", "left", "bottom" and/or "right" as attributes. There is also a "Sticky" value for position, which will behave much like fixed, but sit at first within the page flow then "stick" as the page scrolls. This can give a nice animation effect and is often used for navigation and footer areas in a webpage.

The Float property allows an element to "float" left or right in its container. Try adding a float property to the image in this paragraph (make the penguin less awkward) and see how it sits in line with the text. Even better: use Clearfix to get the next paragraph to clear. Note: images by default are inline-block level elements.

If you have a lot of content inside of a container that you want to have a fixed height and let that content scroll, you can change the Overflow property to "scroll" and set the height of the container to a specific value.
